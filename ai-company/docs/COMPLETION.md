## 🎯 Key Concepts

### Critic Loop
```
Generate Result
    ↓
Score < 90? → Claude criticizes
    ↓
Improve Result → Loop (max 5)
    ↓
Score ≥ 90? → Accept ✓
```

### Specification Verification
```
Implementation → Verify Against Spec
    ↓
100% Complete? → Accept ✓
    ↓
Missing Requirements? → Send back to improve
```

### Persistent Memory
```
Orchestrator reads memory/ before each run
    ↓
Skills execute with full context
    ↓
Decisions/issues saved back to memory/
    ↓
Next run starts with learned context
```

### Task Ordering
```
tasks/request_001/
  01_gather_requirements.md  ← First
  02_write_spec.md           ← Second
  03_design_architecture.md  ← Third
  04_implement_feature.md    ← Fourth
  05_write_tests.md          ← Fifth
```
